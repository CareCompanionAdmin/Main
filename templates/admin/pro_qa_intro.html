Welcome to the Pro QA Workspace

This area is a private workspace for Bryan and the assigned QA tester. Nothing here is visible to the general admin team, parents, or support staff.

What's in each tab

  • Info — Shared Markdown area. Use it to record test account credentials, environment URLs, how to set up TestFlight, and any other reference material that both of us should be able to update freely.
  • Requested Checks — A working list of things to test. Bryan adds items he wants specifically verified or knows are risky; the QA tester marks each as in-review or done as work progresses.
  • Issue Tracker — Private bidirectional ticketing. Either of us can open an issue, comment back and forth, attach screenshots, change status, and link parent/child issues. Same data is visible from dev and prod.

Conventions

  • All text fields accept Markdown. Use the Preview tab to verify formatting before saving.
  • Image and PDF attachments up to 20 MB are stored in S3.
  • Use the Environment field on issues to record where you observed the problem (dev vs prod) — both share this workspace.
